# GitHub Cleanup Checklist

**Purpose:** Use this checklist to ensure your GitHub repository is complete, organized, and ready for review.

---

## Repository Structure

-   [ ] All course folders are present and named according to the organization naming structure.
-   [ ] The following technical development folders are included (all lowercase, with underscores):
    -   mock_interviews
    -   ai_utilization
    -   troubleshooting_debugging
    -   coding_practice
-   [ ] Any additional course-specific folders follow the same lowercase_with_underscores naming style.
-   [ ] There are no stray folders or files outside of their proper directory.
-   [ ] A dedicated folder exists for backups or experiments, such as backup/ or tests/.
-   [ ] No unnecessary duplicates (for example, “project_copy” or “final_v2”) remain.

---

## File Organization and Hygiene

-   [ ] No temporary, auto-save, or cache files (for example, .DS_Store, Thumbs.db, .tmp, or ~$filename).
-   [ ] All files use clear, descriptive names—avoid spaces or random strings (for example, final_project.py not proj123.py).
-   [ ] Each folder includes a README.md file that briefly explains its contents.
-   [ ] Delete large or unnecessary binary files (for example, .zip, .exe, .mp4) unless essential for the project.

---

## Version Control Best Practices

-   [ ] Commit history contains clear and meaningful commit messages (for example, “Add API endpoint for login” rather than “stuff”).
-   [ ] No leftover branches named test, tmp, or junk; merge or delete them.
-   [ ] .gitignore file is present and correctly configured (to exclude .env, **pycache**, .vscode, etc.).
-   [ ] Repository visibility settings match requirements (for example, public for evaluation).
-   [ ] Root-level README.md file includes:
    -   Project description
    -   Installation/setup instructions
    -   Technologies used
    -   Author/contact information (optional)
    -   etc.

---

## Professional Presentation

-   [ ] GitHub profile includes a professional profile README (username/username repository).
-   [ ] Profile photo and bio are professional and up to date.
-   [ ] Pin 3–6 top content that best showcase your skills.
-   [ ] Ensure each pinned repository has a clear, descriptive README.md.
-   [ ] Add a LICENSE file to open-source projects (for example, MIT, Apache 2.0).

---

## Final Verification

-   [ ] Repositories build and run successfully (if applicable).
-   [ ] No sensitive information (such as passwords or API keys) appears anywhere in code or documentation.
-   [ ] All links in README.md files are valid and functional.
-   [ ] Any badges or images display correctly.
-   [ ] Conduct a peer review: another student or mentor confirms your GitHub meets these standards.
